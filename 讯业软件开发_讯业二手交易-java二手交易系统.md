# 一、产品概述
本平台为二手物品交易平台，核心流程为：用户（客户端）提交二手商品的详细介绍及图片至平台进行评估；平台后台接收评估订单后，指派给评估师；评估师完成价格评估并反馈给用户；用户接收评估价格后，可选择将商品交由平台或拒绝；若选择交由平台，将安排快递上门取件，平台收到商品后，向用户发放平台专属金币；金币仅可用于平台商城内二手物品的购买，不可兑现。

# 二、功能逻辑
## 2.1 交易模式
采用用户与平台直接交易的模式，不支持用户之间直接交易，确保交易流程可控、物品质量可追溯。

## 2.2 物品流程
用户上传物品信息 → 平台审核并指派评估师进行评估 → 用户确认评估价格后邮寄物品 → 平台接收物品后进行维修保养及重新拍照 → 平台后台将物品上传上线，供其他用户兑换购买。

## 2.3 物品归属
用户将物品交由平台后，物品最终所有权归平台所有，平台有权对物品进行维修、定价、上架等处置。

## 2.4 价格说明
评估价格为初始参考价格，若平台收到物品后，发现存在未审核的破损、污渍等情况，有权对初始评估价格进行调整，调整后将同步告知相关用户。

# 三、板块明细
## 3.1 用户端（微信小程序）功能明细
### 3.1.1 登录相关
分类：微信小程序

一级功能：登录

二级功能：微信授权登录

描述项：为用户提供便捷的登录方式，支持微信授权登录，同时支持通过手机号获取短信验证码完成验证登录，保障账号安全。

### 3.1.2 首页相关
分类：微信小程序

一级功能：首页

二级功能1：轮播图

描述项：以图文结合的形式展示平台重点内容（如活动、热门物品等），提升用户视觉体验。

二级功能2：通知公告

描述项：展示平台发布的各类公告信息，支持用户点击查看公告详情页，详情页以图文形式呈现，确保信息传递清晰。

二级功能3：上传闲置

描述项：用户可在此功能模块提交闲置物品信息，需选择物品类型、输入商品名称及详细文字描述，上传图片（支持拍照上传或从相册选择，可上传3-6张图片）；同时需填写联系人、联系电话，选择所在区域及详细地址，完成信息填写后提交至平台审核评估。

二级功能4：兑换规则

描述项：以文字列表的形式，清晰展示平台金币兑换相关规则，方便用户查阅了解金币使用规范。

### 3.1.3 消息管理相关
分类：无

一级功能：消息管理

二级功能1：消息列表

描述项：展示平台发送给用户的所有消息内容，包括评估反馈、取件通知、系统通知等，方便用户及时查看。

二级功能2：操作

描述项：支持用户对消息进行新增、编辑、删除及排序操作，满足用户个性化消息管理需求。

### 3.1.4 意见反馈相关
分类：无

一级功能：意见反馈列表

描述项：展示用户提交的所有意见反馈内容，包括反馈文字、反馈图片、提交时间、用户昵称及联系电话，便于平台查看处理。

### 3.1.5 小程序内容管理相关
分类：无

一级功能：小程序内容管理

二级功能1：用户协议

描述项：展示平台用户协议的图文详情内容，用户可随时查阅，明确双方权利与义务。

二级功能2：隐私协议

描述项：展示平台隐私协议的图文详情内容，告知用户个人信息收集、使用及保护相关规则，保障用户隐私安全。

二级功能3：公司简介

描述项：以图文形式展示公司相关信息，增强平台可信度，让用户了解平台背景。

二级功能4：金币兑换人民币比例设置

描述项：支持平台设置金币与人民币的兑换比例，具体为1元可兑换对应数量的金币，比例可根据平台运营需求调整。

二级功能5：轮播图

描述项：包含轮播图片及对应的图文详情内容，可用于展示平台活动、优质物品等，点击轮播图可查看详情。

二级功能6：公告

描述项：展示平台发布的各类公告的图文详情内容，方便用户全面了解公告信息。

# 五、架构特性
1. 采用前后端分离架构，支持独立开发，符合当前主流开发模式，提升开发效率与系统可维护性。

2. 前端以Vue3+Vite为主技术栈，搭配AntdV UI界面框架，视觉清新、操作便捷。

3. 后端以SpringBoot3为基础，结合MybatisPlus数据操作框架、Redis缓存框架，保障系统性能稳定。

4. 采用Maven多模块管理，插件化开发模式，方便组件的安装、卸载与升级，降低模块间耦合度。

5. 业务模块与API接口分离抽离，模块之间可便捷引用，提升代码复用性。

6. 数据库设计精巧，字段规范，具备良好的扩展性，可满足平台后续功能迭代需求。

7. 支持国产密码算法加解密，可满足等保测评国产项目需求，适配国产化部署。

8. 支持MYSQL、ORACLE、SQLSERVER、PGSQL等主流标准结构式数据库，同时兼容达梦、人大金仓、南大通用等国产数据库。

9. 支持中创、宝蓝德、东方通等中间件，适配Windows、Linux及国产操作系统部署，兼容性强。

# 六、功能特性
1. 具备完善的系统基础功能，满足平台日常运营使用需求，避免重复开发，提升开发效率。

2. 支持本地文件、阿里云文件、腾讯云文件、MINIO文件上传，适配不同存储需求。

3. 支持本地邮件、阿里云邮件、腾讯云邮件发送，满足平台各类通知邮件推送需求。

5. 采用B、C端双账号认证体系，会话治理各自独立，保障账号安全与系统稳定。

6. 具备完善的审计功能，包括登录日志、操作日志、异常日志等，便于平台追溯操作行为、排查问题。

7. 具备完善的监控功能，包括会话监控、数据源监控、系统监控等，实时掌握系统运行状态。

8. 支持组织机构、权限管理、定时任务、系统配置等基础功能，满足平台精细化运营需求。

# 七、安全特性
1. 采用SaToken轻量级Java权限认证框架，功能强大、学习成本低，保障系统权限管理安全。

2. 支持登录认证、权限认证、单点登录、三方登录、OAuth2.0等多种认证模式，适配不同登录场景。

3. 采用增强的RBAC权限设计，资源与接口独立授权，权限分配更加灵活。

4. 支持按钮级别细粒度独立授权，界面按钮根据用户权限动态展示，提升系统安全性。

5. 支持API接口注解式、路由拦截式鉴权，有效防止越界访问，保障接口安全。

6. 独创的数据范围机制，每个接口可配置不同的数据范围，确保数据访问权限精准可控。

7. 支持限流防抖、防重复提交功能，有效阻止脏数据产生，保障数据一致性。

8. 密码、手机、身份证号等敏感信息采用国密算法加密传输与存储，提升数据安全性。

9. 操作日志使用SM2进行完整性保护，满足安全审计要求，便于追溯操作行为。
